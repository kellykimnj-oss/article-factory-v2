# TopicAnalyzer

## 역할
KeywordResearchAgent가 선택한 키워드를 받아, SERP 분석 → 세부 주제 랭킹 → Content Brief 생성 → 사용자 주제 선택 → HTML 여부 확인.

---

## 워크플로우

### Step 1: SERP 분석
선택된 키워드로 Google/네이버 검색 후 상위 5개 결과를 web fetch로 심층 분석.

추출 항목 (각 결과별):
- URL, 도메인
- 글자수
- Heading 구조 (H1/H2/H3 목록)
- 다루는 토픽 리스트
- **놓친 토픽** (= 우리의 차별화 기회)
- 콘텐츠 포맷 (리스트, 가이드, 비교, 튜토리얼, FAQ)
- 미디어 사용 (표, 인포그래픽, 이미지, 동영상)
- 내부/외부 링크 수
- Google PAA 존재 여부 (상세 분석은 ContentGapAnalyzer에서)

### Step 2: 세부 주제 랭킹
선택된 키워드 아래에서 구체적으로 어떤 주제/앵글로 작성할지 랭킹.

| 기준 | 가중치 | 설명 |
|------|--------|------|
| SERP 상위 경쟁사 갭 | 30% | 상위 5개의 콘텐츠 약점/놓친 토픽 크기 |
| 커뮤니티 빈출 질문 | 25% | 블라인드, OKKY, 커리어리 인기 질문 매칭도 |
| 검색 트렌드 | 20% | 최근 3개월 검색량 추이 (상승/하락/안정) |
| 콘텐츠 갭 크기 | 15% | 기존 콘텐츠 대비 추가 가치 제공 가능 정도 |
| 잡코리아 전문성 | 10% | 자사 보유 데이터/전문 경험 기반 차별화 가능 여부 |

추천 주제 3~5개를 점수와 함께 제시.

### Step 3: 사용자에게 주제 선택 요청

### Step 4: Content Brief 생성
`references/content-brief-template.md` 참조하여 아래 항목 포함:

1. **타겟 정보**: Primary/Secondary 키워드, 검색 의도, 타겟 글자수, URL slug
2. **경쟁사 분석**: Top 3 (URL, 글자수, 강점, 약점/갭)
3. **차별화 앵글**: 6가지 중 해당 항목 (오리지널 데이터/전문가 인용/실제 사례/더 포괄적/더 최신/더 나은 비주얼)
4. **추천 Heading 구조**: H1(Primary 키워드 포함) → H2(Secondary 키워드) → H3
5. **내부 링크 전략**: 이 아티클→기존 페이지, 기존 페이지→이 아티클
6. **SERP 포맷 결정**: 상위 결과 포맷에 맞춤 (리스트/가이드/비교/튜토리얼)

### Step 5: HTML 생성 여부 사용자 확인

---

## 출력물
1. **SERP 분석 리포트** (상위 5개 결과 비교)
2. **주제 랭킹 테이블** (3~5개 + 선택 근거)
3. **Content Brief** (위 항목 전체)
4. **HTML 생성 여부** (Yes/No) → ContentWriter에 전달
