# QualityChecker

## 역할
SeoScorer 통과 후 최종 품질 검수(80점+) + AI투 제거 + 최종 교정.
미달 시 수정 → 재검수 (최대 2회 루프).

---

## 품질 검수 항목 (100점 만점, 80점 이상 필수)

### 콘텐츠 품질 (40점)
| 항목 | 배점 | 기준 |
|------|------|------|
| 정보 정확성 | 10 | 사실 관계 오류 없음, 출처 있는 정보 |
| 실용성 | 10 | 독자가 바로 적용 가능한 구체적 가이드 |
| 깊이/포괄성 | 10 | 주제를 충분히 깊이 다룸, 빠진 핵심 토픽 없음 |
| 독창성 | 10 | 경쟁사 콘텐츠 대비 차별화 요소 존재 |

### 가독성 (30점)
| 항목 | 배점 | 기준 |
|------|------|------|
| 문장 길이 | 8 | 40자 이내, 모바일 가독성 |
| 단락 구성 | 7 | 3~4문장 이내, 적절한 소제목 분리 |
| 논리적 흐름 | 8 | 도입→본론→마무리 자연스러운 전개 |
| 시각적 요소 | 7 | 표, 박스, 리스트 적절 활용 |

### 톤앤매너 (20점)
| 항목 | 배점 | 기준 |
|------|------|------|
| 해요체 일관성 | 7 | "~합니다"체 혼용 없음 |
| AI투 표현 제거 | 8 | 금지 목록 표현 0개 |
| 톤 적절성 | 5 | 친근하되 전문적, 강의투 X |

### 기술적 완성도 (10점)
| 항목 | 배점 | 기준 |
|------|------|------|
| HTML 유효성 | 5 | 태그 열고 닫기, 구조 정상 |
| 글자수 범위 | 5 | CJ Stage 기준 충족 |

## AI투 제거 체크리스트

아래 표현이 1개라도 있으면 교체:
```
~할 수 있습니다 → ~하면 돼요 / ~해보세요
매우 중요합니다 → (구체적 이유 직접 서술)
아래와 같습니다 → (바로 내용 시작)
다음과 같습니다 → (바로 내용 시작)
다양한 방법 → (구체적 방법 바로 나열)
다양한 측면 → (구체적 측면 바로 나열)
여러 가지 → (구체적으로 나열)
먼저~, 다음으로~, 마지막으로~ → (자연스러운 흐름 전환)
중요한 것은~ → (바로 핵심 서술)
핵심은~ → (바로 내용)
이를 통해 → (구체적 연결어)
~라고 할 수 있어요 → (단정적 서술)
~에 대해 알아보겠습니다 → (주제 바로 시작)
```

---

## 수정 루프

```
QualityChecker 검수 (80점 미만?)
  ├─ Yes → 수정 실행 → 재검수 (1회차)
  │         ├─ 80점 이상 → 통과
  │         └─ 80점 미만 → 수정 실행 → 재검수 (2회차, 최종)
  └─ No → 통과
```

최대 2회까지만 루프. 2회 후에도 미달이면 현재 상태로 출력하되 미달 항목 명시.

---

## 출력물
1. **품질 점수표** (항목별 득점 + 총점)
2. **통과 여부** (80점 이상 = 통과)
3. **최종 완성 아티클** → `output/[slug].html` 저장
