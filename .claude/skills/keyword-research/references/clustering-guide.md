# 키워드 클러스터링 가이드

## 원칙
1페이지 = 1 키워드 클러스터. 별도 페이지를 만들면 키워드 카니발라이제이션 발생.

## 클러스터링 방법

### Step 1: Secondary 키워드를 의미 그룹으로 묶기
```
Primary: "백엔드 개발자 이력서"
클러스터 A (작성법):
  - 백엔드 개발자 이력서 작성법
  - 백엔드 이력서 양식
  - 개발자 이력서 쓰는 법
  - 서버 개발자 이력서 예시

클러스터 B (포트폴리오):
  - 백엔드 포트폴리오 이력서
  - 개발자 포트폴리오 없이 이력서
  - 주니어 백엔드 이력서 프로젝트

클러스터 C (경력별):
  - 3년차 백엔드 이력서
  - 경력 백엔드 개발자 이력서
  - 시니어 백엔드 이력서
```

### Step 2: 클러스터 간 중복 확인
- 같은 검색 결과 페이지에 동일 URL이 여러 클러스터에 나타나면 → 하나로 합침
- 검색 결과가 완전히 다른 페이지들이면 → 별도 클러스터 유지

### Step 3: 클러스터별 1개 Primary 키워드 지정
- 클러스터 내에서 검색 수요가 가장 높은 키워드를 Primary로
- 나머지는 본문에서 자연스럽게 커버 (키워드 밀도 1-2%)
